# LocalQL v1 Release Notes

LocalQL v1 provides a local CSV workflow backed by DuckDB SQL. Install the
`localql` package and use the `csvql` command.

## 1.0.5

LocalQL 1.0.5 strengthens package and release verification by checking the
wheel and source distribution as one exact release bundle before publication.

Version-reporting surfaces, including `csvql --version` and
`csvql.__version__`, now report 1.0.5. Query behavior, supported formats, and
the SQL safety boundary remain unchanged.

## 1.0.4

LocalQL 1.0.4 carries forward the 1.0.3 experience without changes to the CLI,
Python API, supported file formats, or SQL safety boundary.

## 1.0.3

LocalQL 1.0.3 improves result handling in the optional terminal menu and keeps
the v1 release documentation current.

### Highlights

- Restored results and unavailable results are handled more reliably in
  `csvql menu`.
- The changelog and release notes provide a clearer guide to the current v1
  experience.

The CLI, Python API, supported file formats, and SQL safety boundary remain
compatible with LocalQL 1.0.1.

## 1.0.1

LocalQL 1.0.1 improves the public documentation and fixes README links and
screenshots in the PyPI package description. It does not change the CLI, Python
API, or supported file formats.

## 1.0.0

### Highlights

- Query one CSV directly or join named CSV tables.
- Save a repeatable project catalog in `.csvql.yml`.
- Run saved SQL and export results as CSV, JSON, Markdown, or text.
- Inspect, sample, and profile sources, and run configured data-quality checks.
- Use `csvql menu` as an optional terminal workbench.

### Core CLI and optional terminal menu

The core `csvql` command is installed with `localql`. The terminal menu is an
optional extra installed with `localql[tui]`. It provides a source list, SQL
editor, results, and session history; the same CSV sources and SQL safety rules
apply to both interfaces.

`F4` or `Ctrl+R` runs selected SQL or the current statement. `F12` or
`Ctrl+B` runs a semicolon-delimited buffer. See the
[Terminal menu guide](../tui-guide.md) for the full key reference.

### Compatibility and safety

LocalQL supports Python 3.11 through 3.14 on macOS, Linux, and Windows.

LocalQL executes trusted local DuckDB SQL. It does not sandbox DuckDB or
restrict filesystem access. Run only SQL you trust.
