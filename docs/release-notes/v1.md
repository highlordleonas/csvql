# LocalQL v1 Release Notes

## 1.0.2

- Hardened the optional terminal workbench's large-result storage with secure
  operating-system temporary workspaces, atomic spill completion,
  conservative abandoned-workspace recovery, bounded previews, and sanitized
  cleanup or storage-failure reporting.
- Added portable control-key fallbacks for terminal workbench actions where
  terminals or operating systems intercept function keys.
- Sharpened installed-user documentation across pip and uv-tool paths.
- Validated separate core and optional TUI installation paths.
- Confirmed the v1 CSV CLI, Python, catalog, JSON, exit-code, and export
  contracts are unchanged.

TUI spill storage reduces duplicate retained result memory after handling, but
query execution still fully materializes Python results and is not streaming or
execution-memory bounded.

## 1.0.0

LocalQL 1.0.0 lets you query local CSV files with DuckDB SQL. Install the
`localql` package and use the `csvql` command.

### Highlights

- Query one CSV directly or join named CSV tables.
- Save a repeatable project catalog in `.csvql.yml`.
- Run saved SQL and export results as CSV, JSON, Markdown, or text.
- Inspect, sample, and profile sources, and run configured data-quality checks.
- Use `csvql menu` for an optional terminal workbench.

### Terminal menu

The optional terminal menu provides a source list, SQL editor, results, and
session history. `F4` or `Ctrl+R` runs selected SQL or the current statement;
`F12` or `Ctrl+B` runs a semicolon-delimited buffer. The active result can be
exported or saved as a CSV source. See the [Terminal menu guide](../tui-guide.md)
for the full key reference.

`Tab` opens SQL completion when suggestions are available; otherwise it inserts
four spaces. `Ctrl+Space` is an alternative completion key when the terminal
delivers it. Generated SQL is editable and never runs automatically.

### Compatibility and safety

LocalQL supports Python 3.11 through 3.14. It is a local tool for trusted
DuckDB SQL and does not sandbox DuckDB or restrict filesystem access. Run only
SQL you trust.

For command examples and JSON shapes, use the [CLI reference](../cli-reference.md)
and [JSON output reference](../json-contracts.md).
